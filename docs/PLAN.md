# 项目执行计划（核心链路版）

## 计划挑战与收敛原则

### 核心业务链路（唯一优先级）
- [ ] 登录成功（`user/password`）并进入看板
- [ ] 加载看板数据并完成新增/编辑/拖拽
- [ ] 变更可持久化，刷新后不丢失
- [ ] AI 对话返回 `reply`，可选 `board_update` 安全落库并触发前端刷新

### 测试策略（高价值优先）
- [ ] 以“单元测试 + 少量契约/集成 + 关键 e2e”分层，避免同一风险在多层重复验证
- [ ] 每个 Part 只保留 1-3 个能阻断主链路风险的测试点
- [ ] 避免低价值用例：纯样式快照、机械 CRUD 全排列、重复的 UI 文案断言
- [ ] e2e 仅覆盖关键用户旅程，不把细枝末节放进 e2e
